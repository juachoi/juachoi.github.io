Git 기본 명령어 정리

메인 페이지로 돌아가기

저장소 상태 확인

git status
현재 저장소의 상태를 보여주는 명령어입니다.

사용자 설정

git config --local user.name "유저닉네임"
git config --local user.email "유저이메일"
현재 저장소의 작업자를 설정하는 명령어입니다.
깃허브 이메일과 동일하게 설정하면 깃허브에서 작업자 프로필이 연동됩니다.
참고: --local 대신 --global을 사용하면 해당 PC 전체에 설정이 적용됩니다.

원격 저장소 연결

git remote add origin <저장소url>
로컬과 깃허브 저장소를 연결하는 명령어입니다. origin은 저장소를 가리키는 별칭입니다.

저장소 복제

git clone <저장소url>
깃허브 저장소의 내용을 로컬에 복제합니다.
명령어 끝에 '.' 을 추가하면 현재 폴더에 바로 다운로드됩니다.

변경사항 스테이징

git add <파일경로>
git add .
변경사항이 있는 파일을 스테이징 영역에 추가합니다.
'git add .' 는 모든 변경사항을 한 번에 스테이징합니다.

커밋하기

git commit -m "커밋메시지"
스테이징된 변경사항을 커밋메시지와 함께 버전으로 확정합니다.

변경사항 푸시

git push origin main
git push
로컬의 변경사항을 깃허브 저장소에 업로드합니다.
연결된 원격 저장소가 하나라면 'git push'만으로도 충분합니다.

변경사항 가져오기

git pull
깃허브 저장소의 최신 변경사항을 로컬로 가져옵니다.
