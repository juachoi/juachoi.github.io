git status
: 현재 저장소의 상태를 보여주는 명령어

※ 사용자 설정
git config --local user.name "유저닉네임"
git config --local user.email "유저이메일"
: 현재 저장소의 작업자를 설정하는 명령어
이때, 유저 이메일을 깃허브의 이메일과 동일하게 설정하면
기서브에서 작업자 프로필이 연동된다. 

+ --local 대신에 --global을 입력하면 
해당 PC의 어느 위치에서도 같은 설정으로 작업할 수 있다. 

git clone <저장소url>
: 현재 폴더 아래에 저장소 이름의 폴더를 생성하고,
그 아래 파일들을 다운로드 받는다. 
+ 마지막에 공백과 함께 마침표(.)를 적으면 현재 폴더에
파일을 바로 다운로드 받는다.

git add <파일경로>
: 변경사항이 있는 파일을 하나의 버전에 포함시키기 위해
임시로 담는 행동-> staging

버전 확정
git commit -m "커밋메시지"
: 해당 버전에 커밋메시지와 함께 버전을 확정 짓는 것이다. 

git push origin main
git push
: main 브랜치의 변경사항을 깃허브 저장소에 업데이트를 올리는 것
만약에 연경된 깃허브 저장소가 하나라면, origin main은 생략해도 된다. 

git pull
: 깃허브 저장소에 올라온 업데이트 내용을 확인하고 로컬로 받아오는 명령어
